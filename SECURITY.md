# Security Policy

## Supported Versions

Security updates are applied to the latest `0.1.x` release and the current
default branch.

## Reporting A Vulnerability

Use GitHub's **Private vulnerability reporting** feature in the repository
Security tab. Include the affected version, reproduction steps, impact, and any
suggested mitigation. Please avoid opening a public issue for an unpatched
security problem.

Reports covering checkpoint loading, compressed-container parsing, resource
exhaustion, unsafe deserialization, or accidental disclosure of data and secrets
are in scope.
